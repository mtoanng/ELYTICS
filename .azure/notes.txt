General:
	- JSON used for all changelogs

scripts:
	- update_unreleased.py (prevent dupes, only PR's into develop)
	- release_versions.py

pipelines:
	- watch develop -> update changelog to include PR title etc
	- appends PR titles to corresponding changelog.json
	- e.g. { 'unreleased': ['feat(sherlock) Added new polecurve analysis page', ...] }

release pipelines:
	- manual trigger major -> release all unreleased changes to (x+1).y.z
	- manual trigger minor -> release all unreleased changes to x.(y+1).z
	- manual trigger patch -> release all unreleased changes to x.y.(z+1)
	- empty? if no changes in changelog of that space, don't bump version

for the release pipelines, i want to configure this to be manually selected, so a maintainer has to select type of release for each space and root app
once all filled, the release job runs, takes this info and then runs release_versions.py script that updates all the changelogs and merges develop to main, then finally does usual docker build/push
